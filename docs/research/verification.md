**Correction:** aarch64-pc-windows-msvc is Tier 1 with host tools since Rust 1.91.0 (2025-10-30) — correct. Zed's Windows release renders with DirectX 11 (custom GPUI backend, not wgpu). Adreno on Windows ARM64: Qualcomm historically shipped no Vulkan driver at all; the "translation layer over DX12" is Microsoft's Mesa/Dozen Compatibility Pack, and current Snapdragon X drivers include a native Vulkan 1.3 driver that the Compatibility Pack can silently replace with the limited DX12 wrapper. **Evidence:** (1) Rust: CONFIRMED. Rust 1.91.0 (released 2025-10-30) "promotes the aarch64-pc-windows-msvc target to Tier 1 support" (official Rust blog; RFC 3817, rust-lang/rust PR #145682). ### `osc99-spec` — **CONFIRMED**

**Evidence:** All assertions verified verbatim against the live spec (sw.kovidgoyal.net/kitty/desktop-notifications/) and its source (docs/desktop-notifications.rst in kovidgoyal/kitty on GitHub), which agree exactly. (1) Frame: "<OSC> 99 ; metadata ; payload <terminator>"; "The two semi-colons must always be present even when no metadata is present." (2) Metadata: "colon separated key=value pairs. Every key must be a single character from the set a-zA-Z". (3) Size: "The size of the payload must be no longer than 2048 bytes, before being encoded or 4096 encoded bytes"; chunking via same i= across escape codes with d=0 (hold, not done) / d=1 (done, display); terminal concatenates. (4) e=1 marks base64 (RFC 4648) UTF-8 payload. (5) a=: "The value focus... is the default" (values report/focus, '-' prefix negates). (6) o=: always (default) / unfocused / invisible. (7) u=: 0 low, 1 normal (default), 2 critical. (8) Query: client sends OSC 99 ; i=<id> : p=? ; ST; "A conforming terminal must respond with... i=<id> : p=? ; key=value : key=value"; detection: send query then DA1 — DA1 reply without query reply means unsupported (silence = unsupported). (9) "Buttons are a list of UTF-8 text separated by the Unicode Line Separator character (U+2028)... UTF-8 bytes 0xe2 0x80 0xa8." (10) "Terminals must sanitize ids received from client programs before sending them back in responses, to mitigate input injection based attacks." Implementation notes for gmux: the p=? query requires an i= identifier (explicitly for multiplexer routing), the reply's supported keys appear as colon-separated pairs in the payload slot, and query support was "Added in version 0.36.0" of the spec's reference implementation.
